# コンテキスト更新

このコマンドは重要な決定事項や方針をcontexts/ディレクトリに記録します。

## 実行手順

1. ユーザーに更新内容を確認:
   - 何を記録するか
   - 既存ファイルの更新か、新規作成か

2. docs/contexts/ または contexts/ ディレクトリを確認
   - なければ作成を提案

3. 適切なファイルを更新または作成:
   - 既存の関連ファイルがあれば追記
   - 新しいトピックなら新規ファイル作成

4. 更新内容をユーザーに報告

## 記録フォーマット例

### 既存ファイルへの追記

```markdown
## YYYY-MM-DD 決定事項

- [決定内容1]
- [決定内容2]
- 理由: [決定の理由や背景]
```

### 新規ファイル作成

```markdown
# [トピック名]

## 概要

[このファイルの目的]

## 決定事項

### YYYY-MM-DD
- [決定内容]
```

## 注意

- 次回セッションで参照できる形式で記録
- 古くなった情報は整理・アーカイブを提案
- 日本語で記録
