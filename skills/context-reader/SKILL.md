# Context Reader

セッション開始時にプロジェクトのコンテキストを把握するスキルです。

## 動作

### セッション開始時

1. `docs/contexts/` または `contexts/` ディレクトリを探す
2. ディレクトリ内のすべてのMarkdownファイルを読み込む
3. 内容を理解し、現在のプロジェクト状況を把握する

### 把握する内容

- プロジェクトの目的と方向性
- 重要な決定事項
- 現在進行中のタスク
- 注意すべき制約やルール

## コンテキストの更新

ユーザーと重要な決定をした場合:

1. 該当するcontextファイルを編集
2. または新しいcontextファイルを作成
3. 次回セッションで自動的に反映される

### 更新例

```markdown
# プロジェクト方針

## 2025-01-15 決定事項
- コミュニティプラグインとして公開する方針に決定
- macOSのみサポート（Terminal.app）
```

## 使用方法

- セッション開始時に自動発動
- 「コンテキストを確認して」で明示的に実行
- `/update-context` コマンドでコンテキスト更新

## 注意事項

- contextsディレクトリがない場合は作成を提案
- 古い情報は定期的に整理することを推奨
